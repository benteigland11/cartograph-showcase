About

The hardest line of code to delete is the one you'll write again next month.

The pattern

Every developer keeps a private graveyard of useful code: a retry helper, a slug generator, a date parser, an env loader. These functions get rewritten across projects, drift slowly out of sync, and quietly accumulate the same bugs in three different repos at once.

The standard answer is "publish a package." But packages are heavy. A real package needs a name, a registry account, a versioning scheme, a CI pipeline, a license, a changelog, a README, and a willingness to maintain something forever. For a 30-line function you're not sure you'll keep, that overhead is wildly disproportionate.

So most of this code never escapes its origin repo. It just gets copied.

The fix

Cartograph treats reusable code as a widget: a self-contained directory with source, tests, an example, and metadata. The directory is the contract. There's no build, no bundler, no package.json gymnastics.

Every widget passes through the same validator: tests run, contamination scan blocks project-specific names, coverage is enforced, examples must execute. If it validates, it ships.

The principles

  • One widget, one job. No utility belts. No frameworks pretending to be widgets.
  • No widget depends on another widget. If you need shared logic, copy it in. The point is portability.
  • Every variation point is a parameter. No hardcoded thresholds, colors, sizes, or behaviors.
  • Every dependency is declared. The widget is its own contract — read the metadata, know what you're getting.
  • Validation is a gate, not a suggestion. A widget that doesn't pass doesn't ship.

This site

Everything you've seen — the hero, the search, the lab demos, the navigation, the footer, the toast notifications, the 3D tilt cards, the magnetic buttons — is composed entirely from widgets in the cg/ folder of this repo. Each was installed with cartograph install, not written from scratch.

That's the pitch. Code worth keeping. Write it once, iterate.
